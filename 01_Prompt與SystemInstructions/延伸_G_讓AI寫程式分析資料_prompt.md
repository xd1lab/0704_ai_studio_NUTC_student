# 延伸 G · 讓 AI 寫程式分析資料（延伸主題一・Claude）

> **核心觀念**：資料一多，別叫 AI 用「讀」的算數字，叫它**寫程式**——確定性、可重現、可檢查、可重複使用。語言腦的答案你無法驗證；程式的每一步都可驗。
> **使用時機**：會後自學（延伸主題一）——當資料量大到「逐格拉公式」都嫌慢、而你又必須對數字負責的時候。
> **放哪裡**：Claude（`claude.ai` 或桌面版）**新對話**——先上傳 `合成財報明細.csv`，再把 prompt 整段貼上。

---

## 一、使用環境（先讀這段）

- 工具換成 **Claude**：瀏覽器開 `claude.ai`，註冊登入即可（或由官網下載桌面版）。免費帳號即可完成本章全部操作，額度與功能以當天畫面為準。
- **Claude 沒有溫度滑桿**。前幾個主題靠「鎖低溫」求一致；在這裡，「可重現」改由兩件事保證：**規則寫死**＋**讓程式代替語言腦**。同一支程式跑一百次都是同一個答案——比任何溫度設定都硬。
- 口訣升級：**「小量要一致，鎖低溫；量大要一致，寫程式。」**
- 能否「在對話中直接執行程式」，各帳號畫面不一。若你的畫面能顯示程式與執行結果，照「二、」主 Prompt 走到底；若不能，主 Prompt 的【備援條款】會接住你——改拿完整腳本、上 Google Colab 跑（見「三、」「四、」），結果相同、一樣可驗。以當天畫面為準。
- 上傳一律用合成樣本（`02_樣本資料集/財報數字_樣本/合成財報明細.csv`，髒點清單見同資料夾的 `資料字典.md`）；自己的真實資料要先去識別化。
- 建議操作順序：先跑「五、對照實驗」體感一次，再上「二、」主 Prompt 對照。

## 二、可直接複製的主 Prompt（先上傳 CSV，再整段貼上）

```
你是「資料分析工程助理」。我上傳了一份 CSV（合成財報明細.csv），請你「寫程式」處理它，全程不可用閱讀或心算的方式直接估數字。
【工作流程】依序完成，缺一不可：
1. 先列出你打算執行的「清洗規則清單」（一條一條，含判斷依據）。
2. 寫程式並執行（若無法在對話中執行，見【備援條款】）。
3. 每個清洗步驟回報「這一步影響了幾筆」。
4. 輸出彙總結果。
【清洗需求】
- 第 1 列是 # 開頭的浮水印註解，略過；以第 2 列為欄位標題。
- 金額四欄（營業收入、營業成本、稅前淨利、應納稅額）：全形數字轉半形、移除千分位逗號，轉為數值。
- 單位統一：「金額單位」欄混有「千元／元／仟元」；單位為「元」者金額 ÷1000，全部統一為千元，「仟元」標籤正規化為「千元」。
- 報表日期統一為 YYYY-MM-DD：注意民國年（如 112.09.30 需 +1911）、無分隔（20231231）、美式（3/31/2023）等格式。
- 缺值：保留空白並標記筆數，不可自行補數字。
【彙總需求】三張表：
1. 各產業別平均稅前淨利（千元）。
2. 各年度應納稅額合計（千元）。
3. 各公司各年度營業收入合計（千元），並標出 2023 年合計最高的公司；季別不全者照現有列加總，並在表下註明。
【驗證需求】隨機抽 3 筆，列出「原始值 → 清洗後值」對照，讓我能回原檔核對。
【備援條款】若你的環境無法直接執行程式，改為輸出一支完整可執行的 Python 腳本（pandas，含每步 print 處理筆數），並用五步以內教我在 Google Colab 執行。
```

## 三、可直接複製的備援 Prompt（只要腳本）

```
請寫一支完整的 Python 腳本處理我上傳的 合成財報明細.csv，需求同上（浮水印列、全形、千分位、單位換算、日期五格式、缺值標記、三張彙總表）。
要求：用 pandas；檔名固定為「合成財報明細.csv」，我不改任何一行程式；每個清洗步驟 print「處理了幾筆」；最後 print 三張彙總表與 3 筆抽查對照。
我完全不會寫程式，請在腳本之外，用五行以內說明 Colab 操作步驟。
```

---

## 四、Google Colab 免安裝五步（保證能走的備援路）

1. 開瀏覽器到 `colab.research.google.com`，用 Google 帳號登入。
2. 點「新增筆記本」。
3. 點左側的**資料夾圖示**（檔案），把 `合成財報明細.csv` 拖進去（或點上傳圖示選檔）。
4. 把 AI 給的整段程式碼貼進中間的儲存格。
5. 按儲存格左側的 **▶** 執行，往下捲看輸出。（介面以當天畫面為準。）

---

## 五、對照實驗（建議最先做）

先別急著用主 Prompt。開一條**新對話**，一樣**先上傳 CSV**，貼上這題，把答案抄下來（至少問一次；行有餘力再開第二條新對話問同一題）：

```
（請不要寫程式，也不要用任何工具）直接閱讀我上傳的 CSV，告訴我：哪一家公司 2023 年全年營業收入最高？金額是多少？
```

重點不在它會不會答錯，在**你無法驗證**。三種結局都是教材：
- **兩次答案不同** → 不可重現，這種數字不能進報告。
- **答案相同、甚至剛好正確** → 口說無憑：沒有規則清單、沒有每步筆數、無從抽查，你還是不敢簽名。
- **它拒絕硬算、堅持要寫程式** → 連 AI 都知道量大該寫程式——順水推舟，直接進「二、」主 Prompt。

跑完主 Prompt 後回來對照：第三張彙總表就是同一題的**可驗證版本**——有規則清單、有每步筆數、抽得了樣。

## 六、常見坑

- **坑 1｜它不寫程式，直接口算**：重申【工作流程】第 1、2 條，把 prompt 整段原文再送一次。
- **坑 2｜對話中跑不了程式**：功能有無以當天畫面為準；改走「三、」備援 Prompt 拿腳本，照「四、」上 Colab，結果相同、一樣可驗。
- **坑 3｜Colab 報 FileNotFoundError**：檔案沒上傳成功，或檔名不是「合成財報明細.csv」。
- **坑 4｜給了結果卻沒看到程式**：要求「把程式碼與執行輸出完整貼出」；或乾脆走 Colab 自己跑一次。
- **坑 5｜把程式結果當定案**：抽 3 筆回原檔核對，並用 `資料字典.md` 對筆數——單位為「元」約 12 列、全形約 6 列、缺值 5 列。
- **坑 6｜想拿真實資料練**：只用合成樣本；真實資料一律先去識別化。
