## 一、先看懂 AI Studio 的兩個關鍵欄位

在 `aistudio.google.com`（左側選 **Playground**）畫面右側，有兩個本營最常用的東西：

### 1. System Instructions（系統指令）欄位
- **位置**：畫面**右上方**、模型選單附近，通常標示為「**System instructions**」，點開後是一個大文字框（有時預設收合，點一下展開）。
- **作用**：這是給 AI 的「**角色設定與鐵則**」。它的效力**高於**你在對話框臨時打的字。你在這裡寫下的規範（例如「絕不杜撰法條」），會套用在**整個對話串的每一輪回覆**。
- **白話比喻**：對話框 = 你「當下問的問題」；System Instructions = 你事先簽好的「**工作守則合約**」。合約優先。
- **本營用法**：把「文獻分類器」「蘇格拉底助教」等角色的完整設定，貼進這個欄位（詳見各檔案）。

### 2. Temperature（溫度）滑桿
- **位置**：畫面**右側參數面板**，標示 **Temperature**，可拖曳滑桿（範圍約 0 ~ 2；AI Studio 部分模型上限為 1 或 2）。
  - ⚠ 部分較新模型（如 Gemini 3.1 Flash Lite）的面板改顯示 **Thinking level**、**看不到 Temperature 滑桿**——此時把 Thinking level 設低並主力靠 System Instructions，或改用有溫度控制的模型（如 2.5 系列）。
- **作用**：控制回答的「**發散程度 / 隨機性**」。
  - **低（0 ~ 0.3）**：保守、穩定、可重現、貼近事實。**法規審查、資料分類、表格輸出、財報清洗**都用低溫。
  - **中（0.4 ~ 0.7）**：兼顧正確與流暢，適合**摘要、教學說明、對話助教**。
  - **高（0.8 以上）**：發散、同一題每次輸出都不同——**不可重現**，本營的稅務情境**幾乎都不要用高溫**。
- **一句話口訣**：**「要一致就調低溫，要發散才調高溫；分類審查，鎖低溫。」**

> 其他常見旁邊的參數（本營大多用預設即可）：
> - **Top-P / Top-K**：另一種控制發散的方式，初學者維持預設。
> - **Max output tokens**：回覆長度上限，若輸出被截斷可調高。
> - **Model（模型）**：建議用當前 AI Studio 提供的較新 Gemini 版本（如 Gemini 3.x Flash / 2.5 系列，實際以當天畫面為準）；長文獻交叉解讀選**支援長脈絡（long context）**的版本。

## 二、通用使用注意事項（務必先讀）

1. **AI 會「幻覺」**：它可能**非常有自信地**講出不存在的法條字號、判例字號、統計數字。**凡涉及法律效果、金額、字號，一律回原文 / 法規資料庫核對。**
2. **本營所有「法條、字號、金額」範例皆為「示意」**：投影片與講義中若出現具體條號，一律當作教學示範，**請以現行有效法規（全國法規資料庫、財政部函釋、司法院判解）為準**。
3. **溫度要對症**：事實型任務（審查、分類、表格、清洗）→ 低溫；教學互動 → 中溫。
4. **System Instructions 先設定、再對話**：先把角色守則貼好，再開始問，效果最穩。
5. **敏感資料**：示範請用**去識別化 / 公開**的案例文字，避免上傳真實納稅人個資。
6. **不是「取代專業判斷」，是「加速草稿與檢核」**：AI 產出定位為**初稿與交叉檢查工具**，最終判斷仍由教授 / 專業人員負責。

## 四、建議的自學動線

**上午主題（分類與交叉解讀）**
1. 用 **B** 練 few-shot：只給 2 個例子，AI 就會照做 → 免寫程式的自動分類。規則裡寫死「原文未載明就不可補」——這就是防幻覺的起手式。
2. 用 **C** 把 B 的輸出升級成「可貼 Excel 的表格 / JSON」→ 結構化提示。
3. 用 **D** 上傳英文 PDF 與統計圖，做跨文獻對照與圖文矛盾稽核 → 長脈絡與 Context Caching。

**下午主題（互動與試算表）**
4. 用 **E** 切換角色為「蘇格拉底助教」，體驗 AI「不給答案、只給引導」。
5. 用 **F** 收尾，把上午的分類 / 清洗能力**搬進 Google 試算表**，像拉公式一樣批次處理。

**會後延伸（工具換 Claude）**
6. 用 **G** 體驗「量大要一致，寫程式」：上傳 `合成財報明細.csv`，讓 AI 寫程式一次跑完清洗與彙總——每步回報筆數、可抽查、可重跑。
7. 用 **H** 體驗「開新分身才敢咬」：用不帶舊脈絡的全新對話扮審稿人／第二視角／綜合者，投稿前先被電一輪。

## 五、快速故障排除（Q&A 備用）

- **「AI 開始亂編條號怎麼辦？」** → 檢查溫度是否夠低（≤0.2）、規則是否寫死「原文未載明就不可補」（見 B 檔【規則】）；記住：任何字號都要回法規資料庫核對。
- **「輸出格式跑掉、不是表格？」** → 回到 C，明確指定欄位、順序，並要求「只輸出表格，不要多餘說明」。
- **「上傳 PDF 後 AI 說看不到？」** → 確認檔案已成功附加、模型支援檔案輸入；過大時分批或善用 Context Caching（見 D）。
- **「試算表函式 `=AI()` 沒反應？」** → 該帳號可能無此功能，改走 Apps Script 路徑（見 F）。
- **「Claude 跑不了程式？」** → 請它改輸出一支完整 Python 腳本，貼到 Google Colab 免安裝執行（見 G 檔「三、」「四、」）。
