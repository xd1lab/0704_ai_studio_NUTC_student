# 上午 C · 結構化提示範本（Structured Prompt）：輸出可貼 Excel 的表格

> **對應大綱**：結構化提示範本 Structured Prompt
> **核心觀念**：把 B 的分類結果**再升級**——明確定義「欄位（schema）」與「輸出格式」，讓 AI 產出**可以直接貼進 Excel / Google 試算表的表格**，或**可被系統讀取的 JSON**。
> **使用時機**：上午第三段，緊接 B。示範「從一段散文 → 一張整齊的表 → 貼進試算表」。
> **放哪裡**：整段貼進對話框，或把 schema 規則放 System Instructions。

---

## 一、建議參數

| 參數 | 建議值 | 理由 |
|---|---|---|
| **Temperature** | **0 ~ 0.2** | 格式化輸出要**嚴格照 schema**、欄位對齊、不多不少。低溫最穩。 |
| Model | 較新 Gemini | 對 Markdown 表格 / JSON 格式遵循較佳。 |
| Max output tokens | 調高 | 多筆表格易超長。 |

---

## 二、Schema 定義（每欄說明）

| # | 欄位名稱 | 資料型別 | 說明 / 允許值 |
|---|---|---|---|
| 1 | 案例編號 | 整數 | 由 1 開始遞增（批次時用） |
| 2 | 案由 | 文字 | 一句話 |
| 3 | 稅目 | 文字（受控） | 綜合所得稅 / 營業稅 / 營利事業所得稅 / 遺產稅 / 贈與稅 / 地價稅 / 房屋稅 / 其他 |
| 4 | 爭點 | 文字 | 法律上的認定焦點 |
| 5 | 當事人主張 | 文字 | 訴願人 / 納稅義務人主張 |
| 6 | 稽徵機關見解 | 文字 | 原處分機關認定 |
| 7 | 決定結果 | 文字（受控） | 訴願駁回 / 撤銷原處分 / 部分駁回部分撤銷 / 發回另為處分 / 其他 |
| 8 | 關鍵法條 | 文字 | 原文有載才填；否則填「原文未載明」（**不可編造**） |

**共同規則**
- 缺值一律填 `未提及`（文字欄）；受控欄若無法歸類填 `其他`。
- 「關鍵法條」找不到 → `原文未載明`（禁止自行補條號）。
- 逗號 / 換行：表格內文字**不要**用會破壞欄位的字元；若內容含逗號，JSON 版用字串包住即可，Markdown 版避免用「｜」以外的直線。

## 三-A. 可直接複製的 Prompt（Markdown 表格版）

```
你是稅務救濟文件的結構化整理助理。請把我提供的案例文字，整理成「可直接貼進 Excel 的表格」。

【欄位（固定順序，共 8 欄）】
案例編號 | 案由 | 稅目 | 爭點 | 當事人主張 | 稽徵機關見解 | 決定結果 | 關鍵法條

【規則】
1. 只依提供文字抽取，不可加入未出現的資訊。
2. 「稅目」只能填：綜合所得稅／營業稅／營利事業所得稅／遺產稅／贈與稅／地價稅／房屋稅／其他。
3. 「決定結果」只能填：訴願駁回／撤銷原處分／部分駁回部分撤銷／發回另為處分／其他。
4. 「關鍵法條」原文有載才填，否則填「原文未載明」，禁止自行編造條號。
5. 任一欄找不到內容填「未提及」。
6. 只輸出一個 Markdown 表格（含表頭與分隔線），不要任何前後說明文字。
7. 若我提供多筆（以「---」分隔），每筆輸出一列，案例編號依序遞增。
8. 繁體中文、台灣稅務用語。

【待處理文字】
（把一筆或多筆案例文字貼在這裡；多筆用 --- 分隔）
```

**預期輸出（示意，可直接反白複製貼進 Excel）**

| 案例編號 | 案由 | 稅目 | 爭點 | 當事人主張 | 稽徵機關見解 | 決定結果 | 關鍵法條 |
|---|---|---|---|---|---|---|---|
| 1 | 列報扶養胞弟免稅額遭否准 | 綜合所得稅 | 成年胞弟是否符合受扶養要件 | 胞弟在學經濟未獨立，實由其扶養 | 胞弟成年有收入、未同居共財，不符要件 | 訴願駁回 | 原文未載明 |
| 2 | 取具非交易對象發票扣抵進項遭補稅裁罰 | 營業稅 | 非實際交易對象發票可否扣抵及裁罰 | 確有進貨並付款，不應否准並裁罰 | 憑證非交易對象開立，未盡查證義務 | 部分駁回部分撤銷 | 原文未載明 |

> **貼進 Excel 的小撇步**：反白 AI 輸出的表格 → 複製 → 在 Excel 貼上。若整段擠在一格，改用 Google 試算表（對 Markdown 較友善），或用下方 JSON 版 + 「資料 → 分割文字為欄」。最穩的是下午 F 的試算表整合作法。

## 三-B. 可直接複製的 Prompt（JSON 版）

> 需要「餵給程式 / 匯入系統 / 大量批次」時用 JSON，最不容易因逗號、換行而跑掉欄位。

```
你是稅務救濟文件的結構化整理助理。請把我提供的案例文字轉為 JSON 陣列。

【JSON schema（每個物件固定這些鍵）】
{
  "案例編號": 整數,
  "案由": 字串,
  "稅目": 字串,        // 只能是：綜合所得稅|營業稅|營利事業所得稅|遺產稅|贈與稅|地價稅|房屋稅|其他
  "爭點": 字串,
  "當事人主張": 字串,
  "稽徵機關見解": 字串,
  "決定結果": 字串,     // 只能是：訴願駁回|撤銷原處分|部分駁回部分撤銷|發回另為處分|其他
  "關鍵法條": 字串      // 原文有載才填，否則填 "原文未載明"，禁止編造
}

【規則】
1. 只依提供文字抽取，不可加入未出現資訊；找不到的欄位填 "未提及"。
2. 「關鍵法條」找不到填 "原文未載明"，不可自行補條號。
3. 只輸出合法 JSON（一個陣列，即使只有一筆也用陣列），不要 Markdown 標記、不要說明文字、不要註解。
4. 多筆以 "---" 分隔，逐筆轉為陣列元素，案例編號遞增。
5. 內容用繁體中文、台灣稅務用語。

【待處理文字】
（把一筆或多筆案例文字貼在這裡；多筆用 --- 分隔）
```

**預期輸出（示意）**
```json
[
  {
    "案例編號": 1,
    "案由": "列報扶養胞弟免稅額遭否准",
    "稅目": "綜合所得稅",
    "爭點": "成年胞弟是否符合受扶養要件",
    "當事人主張": "胞弟在學、經濟未獨立，實由訴願人扶養",
    "稽徵機關見解": "胞弟成年有收入、未同居共財，不符受扶養要件",
    "決定結果": "訴願駁回",
    "關鍵法條": "原文未載明"
  }
]
```

> **JSON → Excel**：把 JSON 貼進 Google 試算表可用外掛 / 函式轉表；或請 AI 再補一句「請同時給我對應的 CSV」。教學上通常 Markdown 表格版最直覺。

## 四、缺值 / 多筆 / 批次處理規則（重點整理）

- **缺值**：文字欄填 `未提及`；受控欄（稅目、決定結果）無法歸類填 `其他`；法條找不到填 `原文未載明`。
- **多筆**：輸入用 `---` 分隔，AI 逐筆輸出，`案例編號` 遞增。
- **一致性**：務必寫「**只輸出表格 / 只輸出 JSON，不要多餘說明**」，否則 AI 常在前後加解說，導致貼進 Excel 時多出雜訊列。
- **受控欄位**：把允許值列清楚（本檔已列），能大幅減少分類漂移。
- **超長**：批次太多會截斷 → 分批（例如每 20 筆一批），或調高 Max output tokens。

---

## 五、常見坑
- **坑 1：AI 加了前言後語**（「以下是您要的表格：」）→ 一定要寫「只輸出表格，不要任何說明文字」。
- **坑 2：貼進 Excel 全擠一格**→ 改用 Google 試算表貼、或用 JSON/CSV + 分割文字為欄、或走下午 F 的試算表整合。
- **坑 3：欄位內含逗號 / 換行破壞表格**→ 提醒 AI「表格內文字勿含破壞欄位的字元」；量大時用 JSON 最保險。
- **坑 4：受控值被自由發揮**（例如稅目寫成「所得稅（綜合）」）→ 把允許值列死，並寫「只能填以下其中之一」。
- **坑 5：把幻覺條號當真**→ 老話一句：任何條號回原文 / 官方資料庫核對。
