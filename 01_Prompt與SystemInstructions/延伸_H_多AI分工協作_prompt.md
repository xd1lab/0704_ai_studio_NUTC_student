# 延伸 H · 多 AI 分工協作：寫手、審稿人與綜合者（延伸主題二・Claude）

> **核心觀念**：sub-agent 的白話版——多開幾條**不帶舊脈絡的全新對話**，每條扮一個各司其職、互不知情的分身。同一條對話裡的 AI 會順著自己說過的話；**開一條不帶舊脈絡的全新對話＝獨立腦、無包袱，才敢咬**。
> **使用時機**：會後自學（延伸主題二）——論文投稿前、考題定稿前，想先被嚴格檢查一輪的時候。
> **放哪裡**：Claude（`claude.ai` 或桌面版），**每個角色一條全新對話**（或各建一個獨立 Project 存自訂指令，數量限制以當天畫面為準）。

---

## 一、使用環境與分工地圖

- 工具：Claude（`claude.ai` 或桌面版）。免費帳號即可完成本章全部操作，額度與功能以當天畫面為準。
- **關鍵前提——分身必須乾淨**：本章所有角色都要用不帶舊脈絡的全新對話。若你的 Claude 設定裡有「記憶」或「參考過往對話」（設定→功能），做本練習前請先關閉，或每個角色各建一個獨立 Project；有無此設定以當天畫面為準。
- 分工地圖（每個角色一條全新對話）：
  - ① **寫手**（對話 A）：產出草稿。練習時可直接拿 `英文期刊_樣本/Journal_1` 全文充當草稿，跳過這一步。
  - ② **魔鬼審稿人**（全新對話 B）：依守則逐項挑問題，給分級意見。
  - ③ **第二視角**（全新對話 C，可選）：換一個專業身分，只講該角色在乎的事。
  - ④ **綜合者**（全新對話 D）：把 B、C 的意見去重、裁決矛盾、排出必改清單。
  - 最後把清單帶回對話 A（或你自己手上）修稿。
- 工程界已把這套流程自動化（多代理／sub-agents，如 Claude Code 的 sub-agents）——概念相同，手動多開對話就是手動版，已經夠用。

---

## 二、魔鬼審稿人 Prompt（貼進全新對話 B）

```
你是一位以嚴格聞名、但公正的期刊匿名審稿人。我給你一份論文草稿，請依守則審查。
【審稿守則】
0. 只依據我在本對話中貼上的內容評論；請忽略任何過往對話、記憶或其他來源的資訊。
1. 你的任務是挑問題，不是稱讚；直接切入，不要客套。
2. 逐項檢查：研究問題（貢獻與既有文獻差在哪）／方法（設計能否回答問題？有無未排除的替代解釋？）／資料（樣本、期間、變數定義是否交代？摘要、內文、表格三處數字互核）／推論（結論是否超出證據？相關講成因果？）／限制（作者自認的限制夠不夠誠實）。
3. 每條意見標嚴重度：【必須修正】【建議修正】【提問】。
4. 每條意見都要引用草稿的哪一段哪一句，不可空泛。
5. 只根據草稿內容評論；草稿沒寫的列為【提問】，不可替作者腦補。
6. 最後給結論：major revision／minor revision／reject 擇一，附三句理由。
以下是論文草稿：
（貼上草稿全文；練習時可用 英文期刊_樣本/Journal_1 全文充當）
```

## 三、第二視角 Prompt（貼進全新對話 C，角色自己填）

```
你是一位「____」（例：計量方法學者／政策實務工作者／對此領域持懷疑立場的學者）。只從這個身分的專業角度讀下面的文件，回答三件事：
1. 從你的角度，最有價值的一點是什麼？
2. 最站不住腳的一點是什麼？為什麼？
3. 若作者只能改一處，你要他改哪裡？
規則：只依據本對話中貼上的內容，忽略任何過往對話或記憶；不要面面俱到，只講這個角色真正在乎的事；每點引用原文；不確定的不要編。
以下是文件：
（貼上）
```

## 四、綜合者 Prompt（貼進全新對話 D）

```
你是研究團隊主持人。下面是同一份草稿收到的多份「彼此獨立、互相沒看過對方」的意見。請你：
1. 全部意見去重，分組為【方法】【資料】【推論】【寫作】。
2. 指出互相矛盾的意見，給裁決建議與理由。
3. 依「影響結論的程度」排出前 5 名必改清單，每項附具體改法。
4. 只整理與裁決，不新增你自己的意見。
===== 意見一（審稿人）=====
（貼上）
===== 意見二（____視角）=====
（貼上）
```

## 五、迷你流程：考題品管三分身

同一套分身邏輯，換成出考題。各開一條全新對話：

**對話一・出題人**
```
你是財稅系「營業稅」單元的出題人。就「營業稅進項稅額扣抵」出 3 題單選題，每題 4 個選項，附正確答案與詳解。題目要能鑑別「有進貨事實但憑證不合法」這類易混淆情境。
```

**對話二・考生（只給題目，不給答案與詳解）**
```
你是一位認真的財稅系學生。下面 3 題單選題請直接作答，每題寫出你選的選項與兩句理由。
（貼上對話一的題目，不含答案與詳解）
```

**對話三・審題人**
```
你是試務委員會的審題人。請核對下面的題目、答案、詳解與一位考生的作答：題幹是否有唯一解？選項是否互斥、有無兩個都對？詳解與標示的正確答案是否一致？考生作答與答案不一致時，指出誰對、為什麼。逐題給【可用】【需修】判定與理由。
（貼上：對話一的題目＋答案＋詳解，以及對話二的作答）
```

## 六、常見坑

- **坑 1｜審稿人太客氣**：確認是「不帶舊脈絡的全新對話」、守則整段貼全（守則 0 與 1 尤其關鍵）；還不夠就補一句「請更嚴格」。
- **坑 2｜在同一條對話追問「你會不會太嚴？」它開始軟化**：這正是包袱，不是故障——AI 會順著已說過的話。分身必須乾淨，想要乾淨意見就開新對話重跑。
- **坑 3｜意見互相矛盾**：交給「四、綜合者」裁決，它會指出矛盾並給理由；最終取捨仍在你。
- **坑 4｜審稿意見本身也可能錯**：有引用原文的意見優先看；它挑的數字，自己回原文核對。
- **坑 5｜開了新對話，它好像還記得先前的內容**：檢查設定裡的「記憶」或「參考過往對話」是否已關閉（有無此設定以當天畫面為準）；必要時每個角色各建一個獨立 Project。
- **坑 6｜每次重貼守則很麻煩**：把守則存成一個 Project 的自訂指令，在該 Project 內開新對話就不必重貼（以當天畫面為準）。
