## 壹、開始前:登入與環境

### 一、如何登入 Google AI Studio

1. 打開瀏覽器（建議 Chrome），在網址列輸入:**`aistudio.google.com`**。
2. 點右上角 **Sign in（登入）**,用您的 **Google 帳號（Gmail）** 登入。若跳出服務條款,勾選同意即可。
3. 登入後,左側選 **Chat** 或 **Create Prompt（建立提示）**,就會進到本營主要使用的畫面。

> **關於 API Key（金鑰）**:上午大部分示範**不需要**自己申請金鑰。只有**主題五（試算表串接）**若要親手做,才需要一組免費金鑰。屆時講師會帶做,或直接發共用連結給您,**現在不必先弄**。

> **卡關就用共用連結**:若您登入不順、或畫面跟講師不同,請直接舉手。講師會給一個**預先建好的共用連結**,點開就能看到設定好的畫面,不影響您跟上進度。

### 二、介面導覽（先認得這兩個東西）

本營從頭到尾最常動的,就是畫面**右側**的兩個東西。先把它們認熟,後面每一主題都會用到。

**1）System Instructions（系統指令）欄位**
- **在哪**:畫面**右上方**、模型選單附近,標示 **System instructions**。有時預設是收合的,**點一下展開**,裡面是一個大文字框。
- **是什麼**:這是給 AI 的「**工作守則合約**」。您寫在這裡的規範（例如「絕不杜撰法條」),會套用在**整段對話的每一輪**,效力**高於**您在對話框臨時打的字。
- **白話**:對話框 = 您「當下問的問題」;System Instructions = 事先簽好的「合約」。**合約優先。**

**2）Temperature（溫度）滑桿**
- **在哪**:畫面**右側參數面板**,標示 **Temperature**,可拖曳（範圍約 0～2,部分模型上限為 1）。
  ⚠ 部分較新模型的面板顯示 **Thinking level** 而非 Temperature 滑桿——此時把 Thinking level 設低,或改選有溫度滑桿的模型(如 2.5 系列)即可。
- **是什麼**:控制回答的「**發散/隨機程度**」。
  - **低（0～0.3）**:保守、穩定、貼近事實。**法規審查、分類、表格、財報清洗**都用低溫。
  - **中（0.4～0.7）**:兼顧正確與流暢,適合**摘要、教學對話**。
  - **高（0.8 以上）**:發散、同一題每次輸出都不同——**不可重現**,稅務的分類/審查情境**幾乎都不要用**。
- **一句口訣**:**「要一致就調低溫,要發散才調高溫;分類審查,鎖低溫。」**

> 其他旁邊的參數（**本營大多用預設**）:
> - **Model（模型)**:用當前 AI Studio 提供的較新 Gemini;主題三要讀長 PDF,請選**支援長脈絡（long context）**的版本。實際名稱以當天畫面為準。
> - **Max output tokens**:回覆長度上限。若輸出被**截斷**（話講一半停掉),把它調高。

### 四、三個貫穿全場的觀念

1. **AI 會「幻覺」**:它可能**非常有自信地**講出不存在的法條字號、判例字號、金額。**凡涉及法律效果、金額、字號,一律回原文 / 官方資料庫核對。**
2. **溫度要對症**:事實型任務（審查、分類、表格、清洗)→ **低溫**;教學互動 → **中溫**。
3. **AI 是「加速草稿與檢核」,不是取代專業判斷**:產出當作**初稿**,最終判斷仍由您負責。

### 主題二 · 海量文字免寫程式自動分類

**【本節目標】** 只給 AI **2 個範例**（few-shot),不寫任何程式,就讓它把訴願決定書 / 逐字稿自動轉成**規整的表格欄位**。

**【要用的檔案】**
- 提示詞:
  - `01_.../上午_B_文獻分類_few-shot範本.md`（複製「**三、可直接複製的 Few-shot Prompt**」整段——它已內建 2 個範例)。
  - `01_.../上午_C_結構化提示範本.md`（複製「**三-A. Markdown 表格版**」的 prompt,把輸出升級成可貼 Excel 的表格)。
- 樣本:
  - `02_.../訴願決定書_樣本/訴願決定書_A_綜合所得稅補稅罰鍰.md`
  - `02_.../訴願決定書_樣本/訴願決定書_B_營業稅漏開發票.md`
  - （延伸)`02_.../訪談逐字稿_樣本/逐字稿_A_中小企業主談稅務遵循困境.md`、`逐字稿_B_會計師談稅務爭議處理.md`

**【逐步操作】**
1. 溫度設 **0.1～0.3**（分類要一致、可重現）。
2. 把 B 檔「三、」的 **Few-shot Prompt 整段**貼進對話框。它已含【規則】+【範例一】+【範例二】,最後留了「換你做（第三筆)」的空位。
3. 打開 `訴願決定書_A_綜合所得稅補稅罰鍰.md`,把它的**「事實/理由/主文」內文**複製,貼到 prompt 最後「換你做（第三筆)」的位置,送出。
4. AI 會**照前兩個範例的格式**,自動吐出七個欄位:案由、稅目、爭點、當事人主張、稽徵機關見解、決定結果、關鍵法條。
5. **看「決定結果」欄**:A 案應歸「**訴願駁回**」。再換 `訴願決定書_B_營業稅漏開發票.md`,它的決定結果是「**部分駁回部分撤銷（發回）**」——用兩案對照,看 AI 分類的一致性。
6. **升級成表格（接 C 檔)**:改貼 C 檔「三-A. Markdown 表格版」的 prompt,把 A、B 兩案內文用 `---` 分隔一起貼進「待處理文字」,它會輸出**一張 Markdown 表格、每案一列**。
7. **貼進試算表**:反白 AI 輸出的表格 → 複製 → 貼進 **Google 試算表**(對 Markdown 較友善)。文字秒變表。

> **關鍵觀念**:您**沒有寫任何程式**,只給了 2 個例子,AI 就會照做。這就是 few-shot——**用示範代替寫程式**。同一份設定,可以處理 2 筆、也可以處理 300 筆。

**【動手練習】（約 5 分鐘,小份量）**
- 二選一:
  - （A)拿 `訴願決定書_B` 自己跑一次第 3～4 步,確認 AI 把「部分撤銷發回」正確歸類。
  - （B)延伸玩逐字稿:把 `逐字稿_A_中小企業主談稅務遵循困境.md` 貼進去,請 AI「**標出受訪者提到的困境類型**（時間壓力、憑證整理、認定不確定性、舉證責任、遵循成本)並各做一句摘要」。看它能不能從口語逐字稿抓出結構。

**【常見問題 / 卡關排解】**
- **AI 幫我「補」了一個法條?** → 這是幻覺高風險區。B 檔規則已寫死「原文沒寫就填『原文未載明』」;若它還是補,提醒它遵守規則,並記住:**任何條號都要回原文核對**。
- **輸出不是表格、前面多了「以下是您要的表格:」?** → 用 C 檔,並確認 prompt 有「**只輸出表格,不要任何說明文字**」。
- **貼進 Excel 全擠在一格?** → 改用 **Google 試算表**貼;或用 C 檔的 JSON 版 +「資料 → 分割文字為欄」;最穩的是主題五的試算表整合。

### 主題三 · 跨文獻與圖表高效交叉解讀 【觀摩為主】

**【本節目標】** 一次上傳多篇英文期刊 PDF → 自動產出**對照矩陣**;再讓 AI 當「圖表稽核員」,抓出**內文與表格數字互相矛盾**之處。

**【要用的檔案】**
- 提示詞:`01_.../上午_D_跨文獻交叉解讀_prompt.md`（「**二、Prompt 1**」= 對照矩陣;「**三、Prompt 2**」= 圖文矛盾稽核)。
- 樣本:
  - `02_.../英文期刊_樣本/Journal_1_VAT_Compliance_SMEs.md`（**乾淨對照組**)
  - `02_.../英文期刊_樣本/Journal_2_Property_Tax_Revenue_含矛盾.md`（**刻意埋矛盾**)
**【逐步操作】**
1. **選對模型**:把模型切到**支援長脈絡（long context）**的較新 Gemini;溫度設 **0.1～0.3**。
2. **上傳文獻**:在對話框用 **＋ / 迴紋針（附加檔案）** 上傳 `Journal_1` 與 `Journal_2` 兩篇。
   > 若您拿到的是 `.md` 而非 `.pdf`,可請講師提供 PDF 版,或直接把兩篇內文貼進對話框亦可。
3. **跑對照矩陣**:貼 D 檔「二、Prompt 1」整段,送出。AI 會產出一張表:篇名、研究問題、研究方法、資料/樣本、主要發現、結論、研究限制,**每篇一列**,並在表後用幾句話點出兩篇異同。
4. **圖文矛盾稽核（本段高潮)**:針對 `Journal_2`,貼 D 檔「三、Prompt 2」,把 Journal_2 內文中**描述 Table 1 / Table 2 的那段文字**貼進 prompt 的【內文】處。要 AI 逐句比對「內文陳述 vs 表格數字」,分類為【一致】/【矛盾】/【無法判斷】,並列出「⚠️ 矛盾清單」。
5. **看它有沒有抓到**:`Journal_2` 內文用「rose / increased / grew / upward」描述,但**表格數字其實逐年下降**,共**3 處**矛盾（比率、總徵收額、課稅筆數方向全相反)。而 `Journal_1` 是**乾淨組**（內文「下降 4.6 個百分點」與表格一致),可拿來對照什麼叫「沒有矛盾」。

> **小訣竅（也是講師會示範的坑)**:AI 有時會「**順著內文**」而漏抓矛盾。可以先讓它「總結內文」,**再**明確要求它「**逐格核對表格數字**」,稽核效果更好。

> **Context Caching（脈絡快取)——本段用觀念理解即可**:如果您有**一大批**文獻要**反覆問很多問題**,可用「脈絡快取」讓 AI **先把文獻讀熟存起來**,之後連珠炮發問都又快又省。適合文獻回顧、判決整理這種「一份大資料、反覆詰問」的工作。（實際啟用方式因平台版本而異,本營以觀念為主。)

**【動手練習】（約 3～5 分鐘,選做)**
- 有上到金鑰 / 能上傳檔案的老師,可自己跑第 4 步,看 AI 對 `Journal_2` 抓出幾處矛盾;抓完再**自己回表格核對一次**,體會「AI 找疑點、人做最終確認」的分工。抓不到全部也沒關係,這正說明「關鍵數字仍以原文為準」。

**【常見問題 / 卡關排解】**
- **AI 說「看不到檔案 / 讀不到」?** → 確認檔案已成功附加、且模型**支援檔案輸入**;PDF 太大先分批,或改貼純文字。
- **它用「常識」腦補、沒讀我上傳的內容?** → prompt 已強調「只根據我上傳的內容」;仍要提醒:引用的數字回原文查。
- **它漏抓了矛盾?** → 先讓它總結內文,再要求「逐格核對 Table 1、Table 2 每個數字」;必要時逐表分開問。

### 主題四 · 蘇格拉底式互動助教

**【本節目標】** 讓 AI 當「**不直接給答案**」的助教,用連續提問引導學生思考、抓出思考漏洞。示範「AI 不是給魚,是教釣魚」。

**【要用的檔案】**
- 提示詞:`01_.../下午_E_蘇格拉底助教_ChatPrompt.md`（複製「**二、可直接複製的 System Prompt**」整段;個案在「**三、附:具體租稅個案**」)。
- 樣本:個案就在 E 檔裡（**個案甲**=綜所稅醫美列舉扣除;**個案乙**=營業稅憑證來源扣抵)。

**【逐步操作】**
1. 溫度設 **0.5～0.7**（教學互動要自然、能追問,但仍守住不給答案)。
2. 把 E 檔「二、」的 **System Prompt 整段**貼進右上 **System instructions** 欄位。
3. 從 E 檔「三、」挑**一個個案**（建議個案乙:阿美的營業稅憑證案),在對話框開頭寫「這是我們今天要討論的個案:」再把個案文字貼上,送出。
4. AI 會**先簡短歡迎、把事實講清楚,然後只問一個引導問題就停**,把球留給你。
5. **以「學生」身分回應,體驗三種反應**:
   - **故意答錯方向**（例如「她真的有進貨,就應該可以扣」)→ 看 AI**不說你錯**,改用**反問**讓你自己發現矛盾。
   - **故意裝卡住**（「我不知道耶」)→ 看 AI **給提示但不給答案**,把大問題拆小。
   - **好好回答**→ 看 AI **肯定 + 深化追問**（換個金額/身分,結論會不會變?)。
6. **要求它直接給答案試試**(「你直接告訴我結論就好」)→ 看它**禮貌拒絕**,改用一個更小的問題把你往前推。這正是本段亮點。

> **請注意**:整場 AI **不會直接宣布「能不能扣抵」**,而是一路用問題,讓學生**自己**講出「有進貨事實 ≠ 憑證合法可扣抵」。這就是蘇格拉底式教學。

**【動手練習】（約 5～8 分鐘,小份量)**
- 換用**個案甲**（醫美雷射能不能列舉醫藥費扣除),您親自扮學生,至少跑 3 個來回:先答錯一次、再裝卡一次、最後把「有收據 ≠ 符合列舉要件」的重點自己講出來,體會被 AI「一路問到懂」的感覺。

**【常見問題 / 卡關排解】**
- **AI 忍不住直接講答案了?** → 通常是溫度偏高或被你一直逼。可略降溫度,或提醒它「請遵守 System Instructions:不直接給答案,一次只問一個問題」。
- **它一次問太多、回得太長?** → 提醒「請一次只問一個核心問題,簡短一點」。
- **它把我引到錯誤觀念?** → 少見,但請以您的專業把關;E 檔守則要求「可不給答案,但不可引導到錯誤」。

### 主題五 · 試算表免程式碼批次處理

**【本節目標】** 把調校好的 AI 串進 **Google 試算表**,像**拉公式**一樣往下拉,批次**清洗財報數字**、**自動分類產業別**。

**【要用的檔案】**
- 提示詞:`01_.../下午_F_試算表批次處理_prompt與公式.md`（含三條路徑說明、Apps Script 範本、以及情境 1～3 的儲存格公式)。
- 樣本:
  - `02_.../財報數字_樣本/合成財報明細.csv`（**含髒資料**:全形數字、千分位逗號、單位混用、5 種日期格式)。
  - `02_.../財報數字_樣本/資料字典.md`（欄位與髒點清單,先讀它)。

**【逐步操作】**

> 本節有三條路徑。**講師會以「路徑③ Apps Script」為主力示範**(最穩、一定能動);若當天您的帳號剛好有 `=AI()` 函式(路徑②),那更簡單。**跟不上就純觀摩**,檔案帶回家可再練。

**Step 0｜先匯入資料**
1. 開一份新的 Google 試算表 → **檔案 → 匯入** → 上傳 `合成財報明細.csv`。
2. 匯入後**刪掉第 1 列的 `#` 浮水印註解列**,讓第 1 列變成真正的欄位標題（公司代號、年度…金額單位、產業別)。

**路徑③｜Apps Script 串接 Gemini（主力,一定能動)**
3. 先取得一組**免費 Gemini API Key**:回 AI Studio → 取得 **API Key**（Get API key)→ 複製。（講師會帶,或發共用金鑰。)
4. 試算表 → 上方 **擴充功能（Extensions)→ Apps Script**。
5. 把 F 檔「四、最小可用範本」那段 **JavaScript 整段貼進去**,並把 `YOUR_GEMINI_API_KEY` 換成你的金鑰。存檔。
6. 第一次執行會要求**授權**,依指示允許。
7. 回到試算表,在結果欄（例如 C 欄)用 `=GEMINI(...)` 呼叫。**先在少數幾格試**,確認會動,再往下拉。

**Step｜跑三個財報情境（F 檔「五、」有可直接複製的公式)**
假設原始資料在 **B 欄**,結果放 **C 欄**,於 `C2` 貼公式後往下拉:
- **情境 1 · 數字清洗**（去千分位逗號、全形轉半形、去單位文字,只留純數字):用 F 檔「情境 1」公式。例如 `１,２３４,５６７ 元` → `1234567`。
- **情境 2 · 產業別分類**（把細產業別歸到產業大類:科技電子/傳統製造/金融/服務民生/生技醫療):用 F 檔「情境 2」公式,對本 CSV 的 `產業別` 欄跑。例如 `半導體` → `科技電子`。
- **情境 3 · 稅務屬性標註**（可全額認列/有限額規定/原則不得認列/需個案判斷):用 F 檔「情境 3」公式。**此為教學示意,結果一律人工覆核。**

> **本 CSV 的三個大坑**（`資料字典.md` 有完整清單),清洗時特別注意:
> 1. **單位不一致(最關鍵)**:多數列是「千元」,約 12 列是「元」,少數寫「仟元」。**直接把不同單位加總或排名會全錯**——單位是「元」者要 ÷1000 換算成千元。
> 2. **金額髒點**:全形數字、千分位逗號(CSV 中已用雙引號包住)。
> 3. **5 種日期格式**:ISO、斜線、美式月/日/年、**民國年（如 `112.09.30`,需 +1911)**、無分隔（`20231231`)——要統一成 `YYYY-MM-DD`。

**路徑② / ①｜若當天帳號可用(加碼,更簡單)**
- **路徑② `=AI()`**:部分帳號可在儲存格直接用類似 `=AI("把這個產業別歸到產業大類:科技電子/傳統製造/金融/服務民生/生技醫療 其中之一,只回類別:" & B2)`,往下拉即可。**並非所有帳號都有**,以你畫面實測為準。
- **路徑① Workspace 內建 Gemini**:在試算表側邊欄用自然語言下指令。能力依方案而定。

**【動手練習】（約 5 分鐘,小份量)**
- 若您已把路徑③接通:在 C 欄用**情境 2** 的公式,對 `產業別` 欄往下拉幾格,看 AI 把細產業別批次歸成產業大類。再抽查 1～2 格結果對不對(這就是「AI 產初稿、人做覆核」)。
- 若沒接通:純觀摩即可,重點是理解「像拉公式一樣批次處理數百筆」的威力。

**【常見問題 / 卡關排解】**
- **儲存格顯示 `錯誤(400/403)`?** → 金鑰沒填/填錯,或模型名稱過期,或還沒授權。檢查金鑰、把模型名稱換成 AI Studio 當前提供的、重新授權。
- **`=AI()` 沒反應?** → 你的帳號可能沒有這個函式(很常見)。改走**路徑③ Apps Script**。
- **整欄一次拉、跑很慢或報錯?** → 一次送太多請求撞到限制。**先少量幾格,再逐步往下拉**;量大分批。
- **改了來源數字,C 欄沒更新?** → Apps Script 自訂函式不會即時重算,重新輸入該公式或重整頁面。
- **安全**:API Key 是機密,別把含金鑰的試算表隨意分享;**教學後請撤換金鑰**。示範一律用**合成/虛構**資料,不要上傳真實個資。

### 延伸主題一 · 資料量大時,讓 AI 寫程式來分析

**【本節目標】** 幾百列以上的數字彙總,不叫 AI 用「讀」的,改請它**寫一支程式一次跑完**——確定性、可重現、可檢查、可重複使用。主題五是把 AI 請進每一格;這裡是請 AI 退到幕後寫工具,量再大一級,就換這招。口訣升級:**「小量要一致,鎖低溫;量大要一致,寫程式。」**

**【要用的檔案】**
- 提示詞:`01_.../延伸_G_讓AI寫程式分析資料_prompt.md`（主 Prompt 在「二、」,備援腳本與 Colab 在「三、」「四、」,對照實驗在「五、」）。
- 樣本:`02_.../財報數字_樣本/合成財報明細.csv` + `資料字典.md`（先讀髒點清單）。

**【逐步操作】**
1. 開新對話,**上傳 `合成財報明細.csv`**,貼 G 檔「五、對照實驗」那一題:直接問它 2023 年全年營業收入最高是哪家、金額多少,把答案抄下來。（它若拒絕硬算、堅持要寫程式,也是重點——連 AI 都知道量大該寫程式,直接進步驟 2。）
2. **再開一條新對話**,同樣先上傳 CSV,貼 G 檔「二、」主 Prompt 整段送出。
3. 看四段輸出:清洗規則清單 → 程式與執行 → 每步影響筆數 → 三張彙總表。（能否在對話中直接執行程式,以當天畫面為準。）
4. **驗證**:抽 2～3 筆回原檔對照「原始值 → 清洗後值」;再用 `資料字典.md` 核筆數——單位為「元」約 12 筆、全形約 6 筆、缺值 5 筆。
5. 跑不了程式?貼 G 檔「三、」備援 Prompt 拿完整 Python 腳本,照「四、」的五步貼上 Google Colab 執行（免安裝,一定能動）。
6. 回頭看步驟 1 抄下的答案:就算數字剛好對,你當時能驗證嗎?沒有規則清單、沒有筆數、無從抽查——**語言腦的答案你無法驗證,程式的每一步可驗**。主 Prompt 的第三張彙總表,就是同一題的可驗證版本。

**【動手練習】**
- 再開一條全新對話重跑一次主 Prompt:兩次的彙總表應**完全一致**——這就是「可重現」。
- 行有餘力:把【彙總需求】改成「各季營業收入合計」再跑一次,體會「程式可重複使用」。

**【常見問題 / 卡關排解】**
- **它不寫程式,直接用讀的回答?** → 重申【工作流程】第 1、2 條,整段 prompt 原文再送一次。
- **對話中跑不了程式?** → 功能有無以當天畫面為準;改走 G 檔「三、」拿腳本、「四、」上 Colab,結果相同、一樣可驗。
- **Colab 報 FileNotFoundError?** → 檔案沒上傳成功,或檔名不是 `合成財報明細.csv`。
- **程式結果可以直接信嗎?** → 不行,一樣要覆核:抽 3 筆回原檔,並用資料字典核對筆數。AI 出初稿,人做最終判斷,全場鐵律不變。

### 延伸主題二 · 多開 AI 分身:寫手、審稿人與綜合者

**【本節目標】** 把「寫的 AI」和「審的 AI」分開,各用一條**不帶舊脈絡的全新對話**、互不知情——投稿前先被 AI 審稿人電一輪。關鍵在:同一條對話裡的 AI 會順著自己說過的話;**開一條不帶舊脈絡的全新對話=獨立腦、無包袱,才敢咬**。主題三教 AI 抓「錯」（數字矛盾）;這裡請 AI 挑「弱」（方法與推論）——連乾淨的 `Journal_1` 也照樣能被電,這正是重點。

**【要用的檔案】**
- 提示詞:`01_.../延伸_H_多AI分工協作_prompt.md`（審稿人在「二、」,第二視角在「三、」,綜合者在「四、」,考題品管在「五、」）。
- 樣本:`02_.../英文期刊_樣本/Journal_1_VAT_Compliance_SMEs.md`（當「你的論文草稿」的替身;也可換成自己**去識別化**後的草稿）。

**【逐步操作】**
1. **先把對話弄乾淨**:若你的 Claude 設定裡有「記憶」或「參考過往對話」（設定→功能）,做本練習前先關閉,或每個角色各建一個獨立 Project;有無此設定以當天畫面為準。
2. 開全新對話 B:貼 H 檔「二、審稿人」+ `Journal_1` 全文,收回一份標了【必須修正】【建議修正】【提問】的分級意見。
3. 開全新對話 C:貼「三、第二視角」,角色填「計量方法學者」,同一份草稿再審一次。
4. 行有餘力再開一條,角色填「政策實務工作者」。
5. 開全新對話 D:貼「四、綜合者」,把 B、C 的意見整份貼進去,拿到去重、矛盾裁決與「前 5 必改」清單。
6. 把清單帶回自己手上（或寫手對話）修稿。
7. **體感實驗**:回到對話 B 追問「你會不會太嚴格?」——看它開始軟化。這就是包袱:同一條對話裡的 AI 會順著已說過的話,所以審查必須用乾淨的新對話。
8. **驗證**:優先看有引用原文的意見;它挑的數字,自己回 `Journal_1` 核對。

**【動手練習】**
- 考題品管三分身（H 檔「五、」）:出題人、考生、審題人各開一條全新對話,跑一輪看審題人抓出什麼。
- 或用 `訴願決定書_A` 跑三視角:法律學者/稽徵實務/納稅人代理人各開一條,再交綜合者彙整。
- 進階:先開對話 A,請 AI 就「數位發票與中小企業稅務遵循」寫一頁研究構想書,再送對話 B 審——跑完整的「寫手 → 審稿人」循環。

**【常見問題 / 卡關排解】**
- **審稿人太客氣?** → 確認是「不帶舊脈絡的全新對話」、守則整段貼全（守則 0 與 1 尤其關鍵）;還不夠就補一句「請更嚴格」。
- **在同一條對話多追問幾句,它就軟化了?** → 這正是包袱,不是故障。想要乾淨的審查意見,就開全新對話重跑。
- **兩個分身意見矛盾,聽誰的?** → 交綜合者裁決,它會指出矛盾並給理由;最終取捨仍在你。
- **開了新對話,它好像還記得先前聊過的內容?** → 檢查設定裡的「記憶」或「參考過往對話」是否已關閉（有無此設定以當天畫面為準）;必要時每個角色各建一個獨立 Project。
- **每次都要重貼守則,很麻煩?** → 把守則存成一個 Project 的自訂指令,在該 Project 內開新對話就不必重貼（以當天畫面為準）。

## 肆、自學路徑與資源

**接下來自己練,建議照這個順序:**
